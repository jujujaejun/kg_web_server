# 프로젝트 소개

이 프로젝트는 **도서관 회원**이 웹을 통해 도서를 예약하고, **로봇**이 예약을 처리하는 시스템을 구축하는 것을 목표로 합니다. 예약된 도서를 로봇이 자동으로 찾아서 이동하고, **실시간 데이터 전송**과 **3D 모델을 활용한 시각적 랜더링**을 통해 로봇의 상태를 직관적으로 확인할 수 있습니다.

[KG Library App](https://kgwebserver-14978e851dae.herokuapp.com)

## 프로젝트 사용 예시

도서관 회원이 웹 애플리케이션에 접속하여 도서를 예약하면, 서버가 해당 도서의 예약 가능 여부를 확인합니다. 예약이 가능한 경우, **로봇에게 최소한의 데이터를 전달**하여 예약을 진행하며, 로봇은 도서관 내 지정된 위치로 이동하여 작업을 수행합니다.

실시간 데이터는 로봇의 **카메라**와 **위치 센서**를 통해 서버로 전송되며, 3D 모델을 활용하여 웹 페이지에서 로봇의 상태를 직관적으로 시각화할 수 있습니다.

## 사용한 기술 스택

| 기술            | 설명                                              |
|----------------|--------------------------------------------------|
| **API**        | 국회도서관 API를 사용하여 도서 정보를 조회하고 예약을 확인합니다. |
| **데이터베이스** | **MongoDB**를 사용하여 도서 예약, 사용자 정보를 관리합니다. |
| **배포 플랫폼** | **Heroku**를 이용하여 웹 서버를 배포합니다.         |
| **MERN 스택**   | **MongoDB, Express.js, React, Node.js**를 사용하여 웹 애플리케이션을 구축합니다. |
| **3D 모델 랜더링** | **Blender**를 사용하여 로봇의 3D 모델을 만들어 웹에서 시각화합니다. |

## 주요 기능

- **도서 예약 확인**: 도서관 회원이 웹을 통해 도서를 예약하면, 서버가 예약 가능 여부를 확인합니다.
- **로봇 명령 전달**: 예약이 확인되면, 서버는 최소한의 데이터를 로봇에 전달하여 예약 절차를 진행하게 합니다. 예) 예약 시간, 책 고유 ID 등.
- **실시간 데이터 전송**: 로봇의 카메라 영상, 위치 정보 등 실시간 데이터를 서버로 전송하여 웹 페이지에서 바로 확인할 수 있도록 합니다.
- **3D 모델 활용**: 로봇의 상태를 3D 모델로 시각화하여, 사용자는 로봇이 현재 어디에 있는지, 어떤 작업을 하고 있는지 실시간으로 파악할 수 있습니다.

## 주요 기술 설명

- **React**: 도서 예약 및 로봇 상태 시각화를 위한 **프론트엔드** 구현.
- **Express.js**: 서버 사이드 로직을 담당하여, **API 라우팅**과 **데이터 처리**를 수행합니다.
- **Blender**: 로봇의 3D 모델을 생성하여 웹에서 실시간 상태를 시각화합니다.
- **MongoDB**: 도서 예약 및 회원 정보를 저장하고 관리하는 데이터베이스 시스템입니다.
- **Heroku**: 클라우드 배포 플랫폼으로, 웹 서버를 호스팅합니다.

## 프로젝트 구조

이 프로젝트는 **프론트엔드**, **백엔드**, **로봇 제어** 파트를 명확히 분리하여 개발하였습니다. 각 디렉터리의 역할은 다음과 같습니다.

### **디렉토리 구조**
1. **/client**: **React**를 사용한 **프론트엔드** 애플리케이션. `src` 폴더 내에는 UI 컴포넌트 및 애플리케이션 소스 코드가 있습니다.
2. **/agent**: **로봇 제어 관련 코드**로, 로봇의 동작을 제어하는 로직이 포함됩니다.
3. **/server**: **서버 사이드** 로직, **API 라우트**, **데이터베이스 모델** 등을 포함하는 디렉터리입니다.

## 기술적 도전과 해결책

### 성능 최적화
이 프로젝트에서 성능 최적화는 시스템의 주요 목표 중 하나였습니다. 특히 로봇이 실시간으로 데이터를 처리하고 이동할 때, **CPU의 명령어 최적화**를 통해 **성능**을 개선하였습니다. 데이터가 큰 경우, **병렬 처리**와 **캐싱**을 사용하여 대기 시간을 줄이고, 데이터 처리 속도를 획기적으로 향상시켰습니다. 

- **문제**: 실시간 데이터 처리에서의 **응답 속도**가 느려지는 문제.
- **해결책**: **병렬화**와 **비동기 처리**를 통해 동시에 여러 작업을 처리, **최소화된 메모리 사용**으로 더 빠른 응답 속도를 확보했습니다.

### 인공지능을 통한 경로 최적화
로봇은 도서관 내에서 **자율적으로** 경로를 학습하고 **작업을 최적화**하기 위해 **인공지능**을 사용합니다. 로봇이 **장애물 회피** 및 **동적인 경로 설정**을 학습하여, 더 빠르고 효율적인 작업을 수행할 수 있도록 하였습니다.

- **문제**: 로봇이 복잡한 도서관 환경에서 **동적 장애물**을 회피하고 **최단 경로**로 이동하는 데 어려움이 있었습니다.
- **해결책**: **딥러닝 기반의 경로 최적화** 알고리즘을 사용하여 로봇이 주어진 환경에서 **자율적으로 경로를 계획**하고, **실시간 장애물 회피**가 가능하도록 개선했습니다. 이 과정에서 **강화 학습** 기법을 활용하여 로봇의 **효율성**을 지속적으로 향상시켰습니다.

## 배경 설명(공장 로봇의 발전 과정)

### 초기 공장 상태 (산업 혁명 이전)
- **상태**: 산업 혁명 이전에는 **수작업**이 중심이었으며, 사람들이 대부분의 작업을 직접 수행했습니다. 생산은 **소규모**였고, **수공예**가 주를 이루었으며, 생산 속도와 효율성이 낮았습니다.
- **문제점**: 생산성 부족, 작업의 **반복성**으로 인한 **피로**, **높은 비용** 등이 문제였습니다.

### 공장의 초기 모습 (산업 혁명 이후)
- **상태**: **산업 혁명**이 일어난 18세기 후반, 기계화와 **대량 생산**이 가능해졌습니다. **증기 기관**을 활용한 기계들이 등장하며 **자동화**가 시작되었습니다.
- **변화**: 대량 생산을 위한 **기계화**가 이루어졌고, 사람은 기계의 **운영자** 역할을 했습니다. 초기 **컨베이어 벨트**와 같은 기계들이 생산 라인에 도입되었으며, **반복적인 작업**은 기계가 대신 수행하게 되었습니다.

### 로봇의 등장과 발전 (1970년대 이후)
- **상태**: **1970년대**에는 최초의 **산업용 로봇**이 등장하였고, **6축 로봇팔**과 같은 기계들이 **정밀 작업**을 처리하기 시작했습니다.
- **변화**: 초기 로봇은 **단순한 작업**을 처리했지만, 시간이 지남에 따라 **AI**와 **센서 기술**의 발전으로 로봇은 **자율성**과 **정밀도**를 갖추게 되었습니다. 이를 통해 **복잡한 작업**도 처리할 수 있게 되었고, 사람과 함께 협력하는 **협동 로봇**도 등장했습니다.

### 현재까지 로봇의 발전
- **상태**: 현대 공장은 **스마트 팩토리**로 발전하면서, **로봇**과 **인공지능**, **IoT** 등이 결합되어 **자율적이고 효율적인 작업**이 가능해졌습니다. 로봇은 **24시간 작업**을 수행하며, 사람은 **관리**나 **창의적 작업**을 주로 담당합니다.
- **변화**: 로봇은 이제 **정밀한 조립**, **용접**, **페인팅**, **포장** 등 다양한 분야에서 사용되며, **산업용 로봇**과 **협동 로봇**, **AGV**, **자율 로봇** 등 다양한 형태로 발전했습니다. 또한, **AI**가 접목되어 **스스로 학습**하고, **환경에 적응**할 수 있는 능력을 갖추게 되었습니다.

### AGV (Automated Guided Vehicle)와 로봇암

#### 1. **로봇암 (6축 로봇팔)**
**로봇암**이 높은 효율성과 정밀도를 제공할 수 있는 이유는 그 **구조와 동작 원리**에 있습니다. 6축 로봇팔은 여러 축을 자유롭게 움직일 수 있어 **다양한 작업**을 수행할 수 있습니다. 각 축의 움직임을 조정하여 **3D 공간**에서 정확한 작업을 할 수 있기 때문에 복잡한 작업을 빠르고 정확하게 수행할 수 있습니다.

##### **정밀한 작업 가능**:
- **모터 제어 기술**: 6축 로봇팔은 각 축에 독립적으로 모터가 장착되어 있어 **정밀한 움직임**을 제어할 수 있습니다. 이로 인해 **정확한 위치 조정**과 **미세한 작업**이 가능합니다.
- **센서와 비전 시스템**: 로봇에 장착된 **센서**와 **카메라 시스템**은 로봇이 작업을 수행하는 동안 주변 환경을 인식하고 실시간으로 조정할 수 있도록 돕습니다. 예를 들어, **비전 시스템**을 사용하여 부품을 인식하고, **자동 조정**을 통해 정확한 위치에 부품을 조립하는 등의 작업이 가능합니다.

##### **속도와 생산성 향상**:
- **고속 모션 제어**: 로봇은 일정한 반복적인 작업을 매우 빠르고 정확하게 수행할 수 있으며, **휴식 시간** 없이 24시간 연속적으로 작업할 수 있습니다. 이로 인해 대량 생산이 가능하고, **생산 속도**가 크게 향상됩니다.
- **작업 자동화**: 사람의 개입 없이 로봇만으로 작업을 처리할 수 있어 **자동화**된 환경에서 **효율적인 작업 흐름**이 가능합니다. 이를 통해 **생산성**을 높이고, **인건비 절감**과 **인적 오류**를 최소화할 수 있습니다.

#### 2. **AGV (Automated Guided Vehicle)**
**AGV**는 공장 내 물류와 자재 운반을 자동화하는 데 사용되며, 그 효율성을 발휘하는 이유는 **자율 주행 기술**과 **경로 인식 시스템** 덕분입니다.

##### **자율 이동 가능**:
- **자율 주행 시스템**: AGV는 **자율 주행** 기술을 통해 정해진 경로를 따라 자동으로 움직입니다. 레이저, 카메라, 자이로스코프 등의 센서를 활용해 환경을 인식하고, 경로를 따라 이동하며, 장애물이나 사람을 피할 수 있습니다.
- **네비게이션 시스템**: AGV는 **정해진 경로** 또는 **자유로운 경로**를 선택하여 이동합니다. **LIDAR** (Light Detection and Ranging) 기술이나 **SLAM** (Simultaneous Localization and Mapping) 알고리즘을 사용해 실시간으로 위치를 파악하고, 환경에 맞는 경로를 계산하여 이동합니다.

##### **효율성 향상**:
- **자동화된 물류 시스템**: AGV는 **자동화된 물류 흐름**을 지원하여 공장 내 부품의 이동, 창고 내 재고 관리 등에서 인력을 대체할 수 있습니다. 이를 통해 **물류 시간**을 절약하고, **작업의 정확성**을 높일 수 있습니다.
- **인력 절감과 비용 절감**: AGV는 사람 대신 자재를 자동으로 운반하므로 **인건비**를 절감할 수 있습니다. 또한, AGV는 **오류를 최소화**하고 반복적인 작업을 처리함으로써 **효율적인 자원 관리**가 가능해집니다.


### AGV (Automated Guided Vehicle) vs AMR (Autonomous Mobile Robot)
| **특징**                     | **AGV**                      | **AMR**                          |
|------------------------------|--------------------------------------------------------|-----------------------------------------------------------|
| **경로 이동**                | **정해진 경로**를 따라 이동                             | **자율적으로 경로 계획** 및 **장애물 회피** 가능          |
| **환경**                     | **예측 가능한 환경**에서 안정적이고 효율적               | **동적이고 복잡한 환경**에서 유연하게 작업 수행           |
| **기술 요구도**               | **비용 효율적**이고 **유지 관리가 간편**                   | 더 높은 **자율성**과 **기술** 요구                        |
| **적합한 작업**               | **단순하고 반복적인 작업**에 적합                        | **복잡한 작업** 및 **서비스 작업**에 적합                  |
| **예시**                      | 물류 창고, 제조 공장에서 물품 운반                      | 복잡한 창고 내 물류 작업, 서비스 로봇                     |

##### AGV를 선택한 이유
- **도서관** 및 **창고**와 같은 **예측 가능한 환경**에서는 **안정성**과 **효율성**이 중요
- **AGV**는 **정해진 경로**를 따라 **높은 안정성**과 **효율성**을 제공
- **비용 효율성**과 **유지 관리의 용이함**으로 AGV가 더 적합
- 반복적인 작업을 안정적으로 처리할 수 있어 **비용 절감**과 **효율적 운영**이 가능

#### 결론
**로봇암**과 **AGV**는 각각의 기술적 특성과 자동화 기능을 통해 **정밀한 작업**과 **효율적인 물류 자동화**를 가능하게 하여, 공장의 생산성을 크게 향상시키고 비용을 절감하는 데 기여합니다. 이러한 로봇들은 공장에서 가장 영향력이 크기 때문에 이 둘을 결합한 AGV 기반 로봇암을 사용 하였습니다.
